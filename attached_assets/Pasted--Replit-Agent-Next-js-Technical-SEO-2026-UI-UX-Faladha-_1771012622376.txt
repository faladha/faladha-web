التصميم لازم يكون قريب جداً من الستايل الموجود في الصورة المرفقة (Landing Page بنفسجي/تدرجات/كروت/ظلال ناعمة/موك أب جوال/حركات خفيفة). لا تنسخ محتوى/أصول محمية، فقط نفس الفكرة والـ layout.

========================================================
A) STACK + قواعد صارمة
========================================================
1) Next.js (App Router) + TypeScript.
2) Tailwind CSS فقط. بدون مكتبات UI ثقيلة.
3) RTL عربي افتراضياً (dir="rtl")، مع دعم LTR لاحقاً بدون كسر.
4) Fonts: next/font باستخدام خط عربي احترافي (IBM Plex Sans Arabic أو Noto Sans Arabic).
5) Performance:
   - لا تستخدم مكتبات تسبب JS ضخم.
   - استخدم next/image و <Image> في كل الصور.
   - Lazy loading للصور تحت الـ fold.
   - لا تستخدم أي hotlink لصور خارجية.
   - الهدف Lighthouse Mobile: Performance/SEO/BestPractices/Accessibility >= 90.
6) Rendering:
   - SSG لكل صفحات المحتوى (weeks/symptoms/blog/tools) قدر الإمكان.
   - SSR فقط لما يلزم (مثلاً بحث داخلي).
   - ممنوع بناء الموقع كـ CSR-only.
7) Deployment Ready:
   - المشروع لازم يكون “Vercel-ready” بالكامل.
   - أضف إعدادات headers/caching المناسبة إن احتجنا (بدون مبالغة).

========================================================
B) SEO 2026 — المطلوب “محكم”
========================================================
نفّذ كل النقاط التالية بدون استثناء:

1) Metadata لكل صفحة عبر generateMetadata():
   - title فريد
   - description فريد
   - canonical صحيح
   - openGraph + twitter
   - alternates جاهزة (حتى لو لغة واحدة الآن)
   - robots meta مضبوط

2) URL Structure ثابتة:
   - /pregnancy/week-1 ... /pregnancy/week-40
   - /symptoms/[slug]
   - /tools/due-date-calculator
   - /blog/[slug]
   - /faq /features /how-it-works /download /about /medical-review /privacy /contact

3) Structured Data JSON-LD (Reusable Component):
   - Organization + WebSite + WebPage للصفحة الرئيسية
   - BreadcrumbList لكل الصفحات الداخلية
   - Article لكل صفحات week و blog
   - FAQPage في صفحات FAQ + داخل الأسابيع عند وجود FAQ
   - SoftwareApplication أو MobileApplication لصفحة التحميل /download
   - MedicalWebPage في صفحات المحتوى الطبية (بدون ادعاءات علاجية)

4) ملفات SEO:
   - /robots.txt يسمح بالفهرسة ويشير إلى /sitemap.xml
   - /sitemap.xml ديناميكي أو مولّد وقت build (يدعم تجزئة لو كبر)
   - /manifest.webmanifest + favicons + apple-touch-icon
   - أضف security headers الأساسية بدون كسر الفهرسة

5) Internal Linking (إلزامي):
   - كل أسبوع يربط: السابق/التالي + 3 أعراض مرتبطة + أداة واحدة + CTA تحميل
   - صفحة /pregnancy تجمع كل الأسابيع مع وصف قصير وروابط
   - صفحة /symptoms تجمع الأعراض + تصنيفات + روابط
   - Breadcrumb UI + Schema لكل الصفحات

6) منع Thin/Duplicate Content (صارم):
   - ممنوع تكرار العناوين أو الـ meta descriptions بين الصفحات.
   - كل صفحة week لازم تحتوي محتوى مختلف فعلياً، ليس مجرد إعادة صياغة.
   - “مُولّد المحتوى” يجب أن يضمن اختلاف: (الأرقام/الحقائق/النصائح/الأسئلة/الأعراض المرتبطة)
   - أضف validator (سكريبت) يفحص:
     - uniqueness للـ title/description
     - طول المحتوى لكل صفحة (>= 900 كلمة)
     - وجود أقسام إلزامية
     - عدم تكرار FAQ نفس النص بين الأسابيع
