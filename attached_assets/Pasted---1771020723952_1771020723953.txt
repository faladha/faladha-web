أريد منك إنشاء تقرير شامل عن المشروع الحالي بدون أي تعديلات على الكود.

المطلوب: تحليل كامل لحالة الموقع الحالية قبل الإطلاق.

لا تقم بأي تحسينات الآن. فقط اجمع معلومات دقيقة واطبعها في تقرير منظم.

==================================================
1) هيكل المشروع الكامل
==================================================
- اطبع شجرة الملفات الكاملة (file tree)
- حدد:
  - مجلدات app/
  - مجلدات components/
  - مجلدات content/
  - مجلدات api/
  - مجلد admin/
  - ملفات db/schema
  - ملفات scripts (audit / validate)
  - ملفات SEO (sitemap, robots, rss)

==================================================
2) قائمة كل Routes
==================================================
اطبع قائمة بكل الصفحات القابلة للوصول:

- /
- /tools/*
- /pregnancy/*
- /symptoms/*
- /blog/*
- /admin/*
- /api/*

لكل Route:
- هل SSG / SSR / ISR / CSR
- هل فيه metadata
- هل فيه JSON-LD
- هل فيه canonical
- هل فيه internal links
- حجم الصفحة التقريبي (إن أمكن)

==================================================
3) تحليل صفحة الحاسبة
==================================================
- عدد الكلمات
- عدد H2
- عدد FAQ
- هل فيها SoftwareApplication schema
- هل فيها روابط داخلية للأسابيع
- هل فيها جدول 40 أسبوع
- هل فيها CTA
- هل فيها hydration errors

==================================================
4) تحليل الأسابيع 1-40
==================================================
- عدد الصفحات الفعلي
- متوسط عدد الكلمات
- هل كل صفحة فيها:
  - FAQ
  - روابط السابق/التالي
  - روابط أعراض
  - schema Article
- هل هناك تكرار titles أو descriptions

==================================================
5) تحليل الأعراض
==================================================
- عدد الصفحات
- متوسط طول المحتوى
- وجود FAQ
- وجود MedicalWebPage schema
- وجود روابط للأسابيع

==================================================
6) تحليل المدونة
==================================================
- عدد المقالات المنشورة
- متوسط طول المقال
- عدد التصنيفات
- عدد الوسوم
- هل هناك مقالات بدون روابط داخلية
- هل هناك orphan pages
- هل RSS يعمل

==================================================
7) تحليل الداشبورد
==================================================
- هل CRUD يعمل لكل:
  - blog
  - weeks
  - symptoms
  - tools
- هل هناك RBAC
- هل هناك CSRF protection
- هل هناك authentication middleware
- هل هناك مشاكل أمنية محتملة

==================================================
8) SEO Infrastructure
==================================================
- هل sitemap.xml يحتوي كل الروابط المنشورة
- هل lastmod يتحدث تلقائيًا
- هل robots.txt صحيح
- هل هناك duplicate canonical
- هل هناك صفحات noindex بالخطأ

==================================================
9) Performance Snapshot
==================================================
- حجم JS bundle الكلي
- عدد requests في الصفحة الرئيسية
- عدد requests في صفحة الحاسبة
- هل هناك large images
- هل هناك unused JS
- هل font loading optimized

==================================================
10) ملخص نهائي
==================================================
اطبع:

- نقاط القوة
- نقاط الضعف
- المخاطر المحتملة
- مستوى الجاهزية (٪)
- هل جاهز للإطلاق أم لا (Yes/No مع السبب)

لا تقم بأي تحسينات الآن.
فقط اعطني تقرير شامل وموضوعي.
