أنت Replit Agent خبير Next.js App Router + Technical SEO 2026 + UI/UX. 
مهمتك: بناء موقع “فلذة” (Faladha) بهدف #1 تصدر Google لكلمات “حاسبة الحمل” ومشتقاتها + “متابعة الحمل أسبوعاً بأسبوع”، مع تصميم قريب جداً من الصورة المرجعية (تدرج بنفسجي، كروت بيضاء، ظلال ناعمة، موك أب جوال، حركات خفيفة). 
لا أريد أي حلول شكلية أو CSR-only. أريد SSG/SSR صحيح، Metadata + Schema + Robots + Sitemap + RSS + Linking داخلي + مدونة + صفحات أسابيع 1-40 + صفحات أعراض + أدوات. 
نفّذ كل شيء “محكم للسيو” واطبع تقارير الجودة في النهاية.

========================================================
0) قيود عامة (إلزامية)
========================================================
- Next.js (App Router) + TypeScript.
- Tailwind CSS فقط. بدون UI libraries ثقيلة.
- RTL عربي افتراضياً (dir="rtl")، خط عربي عبر next/font (IBM Plex Sans Arabic أو Noto Sans Arabic).
- استخدام next/image لكل الصور (صور placeholders داخل /public).
- لا روابط صور خارجية hotlink.
- الهدف: Lighthouse Mobile >= 90 (Performance/SEO/Best Practices).
- صفحات المحتوى SSG قدر الإمكان. لا تعتمد على CSR.
- لا تكرر Titles/Descriptions بين الصفحات. ممنوع Thin Content.
- اجعل المشروع جاهز للنشر على Vercel (حتى لو التطوير على Replit).

========================================================
9) Validators (Quality Gates) — إلزامي
========================================================
أضف سكريبتين:

1) npm run validate:seo
يفحص:
- uniqueness للـ metaTitle/metaDescription لكل الصفحات
- عدم وجود صفحات بدون canonical
- وجود schema في صفحات: home/tools/week/blog
- طباعة تقرير مختصر

2) npm run validate:content
يفحص:
- week pages >= 900 كلمة
- blog posts >= 900 كلمة
- وجود >=4 عناوين H2 في المقال
- وجود FAQ في الصفحات المطلوبة
- وجود >=3 internal links (weeks/symptoms/blog/tools)
